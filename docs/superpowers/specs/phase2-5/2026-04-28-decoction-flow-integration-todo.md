## 背景

煎药小游戏 UI 已完成并通过 AI 验收（86.5分），但游戏逻辑与整体流程的串联存在多个待确认的设计细节。这些细节涉及：

- 背包系统与煎药环节的数据联动
- 煎药游戏的独立玩法 vs 问诊流程绑定
- 生成药剂的去向追踪

**决策原则**: 等问诊游戏（辨证选方）重构整合后，再整体设计这些流程串联逻辑。

## TODO 列表

### 1. 药材来源与背包联动

| 问题 | 当前状态 | 待确认设计 |
|-----|---------|----------|
| 药材卡片来源 | 使用硬编码方剂数据 | 是否使用背包中的真实药材卡片？ |
| 药材数量消耗 | 不跟踪 | 煎药成功后是否消耗背包药材？数量如何扣减？ |
| 药材不足处理 | 未设计 | 背包药材不足时如何提示？是否阻止煎药？ |

**设计选项**:
- **方案A**: 煎药仅作为"配伍练习"，不消耗真实药材，生成的药剂为虚拟物品
- **方案B**: 煎药消耗背包药材，生成真实药剂（需追踪数量）
- **方案C**: 两模式并存：练习模式（虚拟）+ 生产模式（真实消耗）

| 问题 | 当前状态 | 待确认设计 |
|-----|---------|----------|
| 药剂显示位置 | 仅在煎药界面展示动画 | 是否添加到背包？ |
| 药剂属性 | 无详细属性 | 是否记录方剂名、功效、保质期等？ |
| 药剂用途 | 未设计 | 可用于问诊环节"给病人用药"评分？ |

**设计选项**:
- **方案A**: 药剂仅作为视觉反馈，不进入背包
- **方案B**: 药剂进入背包，作为可查看物品
- **方案C**: 药剂进入背包，且可用于后续问诊环节的"疗效评分"

### 3. 煎药游戏独立玩法

| 问题 | 当前状态 | 待确认设计 |
|-----|---------|----------|
| 进入方式 | 仅通过问诊流程 | 是否允许玩家主动进入练习？ |
| 方剂来源 | 问诊选方环节传入 | 独立玩时如何获取方剂？ |
| 方剂范围 | 未限制 | 是否限定"当前诊所内有方剂"？ |
| 评分意义 | 仅练习反馈 | 独立玩是否计入经验值？ |

**设计选项**:
- **方案A**: 煎药仅作为问诊流程的一部分，不可独立进入
- **方案B**: 允许独立进入，随机出方剂（限定诊所方剂范围），作为配伍练习
- **方案C**: 两种入口并存：
  - 问诊流程：使用选方环节的方剂，产出真实药剂
  - 练习入口：随机方剂，虚拟练习，不计消耗

**UI入口设计**:
- 诊所场景：D键进入煎药
- 独立入口时弹出"方剂选择"界面（从诊所方剂列表随机/手动选择）

### 4. 问诊流程与煎药环节衔接

| 问题 | 当前状态 | 待确认设计 |
|-----|---------|----------|
| 数据传递 | 未实现 | 辨证选方环节如何将方剂传给煎药？ |
| 流程顺序 | 各游戏独立 | 问诊→辨证→选方→煎药的完整链路？ |
| 药剂回用 | 未设计 | 煎药产出的药剂如何用于"疗效评估"？ |

**完整流程设想**:
```
问诊环节 → 收集症状/脉象/舌象
    ↓
辨证环节 → 识别证型
    ↓
选方环节 → 选择方剂（传入煎药）
    ↓
煎药环节 → 配伍/火候评分 → 生成药剂
    ↓
疗效评估 → 药剂+证型匹配度 → 最终评分
```

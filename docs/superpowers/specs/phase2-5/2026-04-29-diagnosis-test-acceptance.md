## 三、AI视觉验收标准

### 评估维度

| 维度 | 权重 | 评估标准 |
|------|------|---------|
| 中医风格 | 20% | 古朴典雅水墨风格，宣纸色调背景 |
| UI布局 | 20% | 左侧导航220px，主内容区1220px，比例正确 |
| 配色一致性 | 15% | --paper/--ink/--accent/--seal色系协调 |
| 文字可读性 | 10% | 字体清晰，无重叠，间距合理 |
| 印章效果 | 10% | 印章位置正确，动画流畅 |
| 舌象SVG | 10% | 舌形清晰，颜色符合描述 |
| 导航交互 | 15% | active状态明显，hover效果流畅 |

### 视觉验收流程

1. **截图捕获**: Playwright截取各阶段完整页面
2. **AI分析**: 多模态模型评估视觉质量
3. **评分输出**: 各维度得分 + 总分
4. **问题标记**: 识别不符合标准的元素
5. **改进建议**: 生成具体修改建议

## 四、测试执行流程

### Phase 1: E2E自动化测试

```bash
# 运行诊断游戏测试
npx playwright test tests/e2e/diagnosis-html-ui.spec.ts --workers=1

# 预期输出
- 通过: ≥12个测试
- 失败: ≤3个测试（可接受的viewport问题）
```

### Phase 2: AI视觉验收

使用 webapp-testing 技能：
1. 启动游戏，进入诊断场景
2. 截取舌诊/脉诊/问诊/辨证/选方各阶段截图
3. AI评估各阶段视觉质量
4. 生成验收报告

### Phase 3: 集成验证

```bash
# 运行完整测试套件
npx playwright test --workers=1

# 验证TypeScript编译
npx tsc --noEmit
```

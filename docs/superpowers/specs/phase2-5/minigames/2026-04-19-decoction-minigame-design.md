## 一、游戏概述

### 1.1 游戏定位

煎药是药园生产流程的核心环节（种植→炮制→煎药），玩家根据方剂要求，完成药材选择、配伍放置、顺序设置、火候控制、煎煮操作，学习中药煎煮规范。

### 1.2 核心玩法

| 玩法要素 | 描述 |
|---------|------|
| **学习目标** | 掌握方剂组成、煎药顺序、火候控制 |
| **操作方式** | 统一场景：左侧煎药炉灶动画+右侧背包药材选择 |
| **难度设计** | 初级(提示组成)→中级(无提示)→高级(自定义加减) |
| **反馈机制** | 评分维度展示(组成/火候/时间)，错误详情 |

### 1.3 游戏时长

- 单次煎药: 2-3分钟
- 完整流程(含煎煮动画): 3-5分钟

## 二、与其他游戏关联

### 2.1 上游依赖

| 来源 | 数据依赖 | 说明 |
|-----|---------|------|
| 选方 | prescriptionId | 煎药目标方剂 |
| 背包 | herbs库存 | 可用药材来源 |
| 炮制 | processedHerbs | 炮制后药材入库 |
| 病案 | decoctionParams | 煎药参数要求 |

### 2.2 下游输出

| 输出数据 | 目标 | 说明 |
|---------|-----|------|
| decoctionResult | 背包 | 煎好的药物入库 |
| score | 经验值 | 获得煎药经验 |
| feedback | NPC | 青木先生评述 |

### 2.3 数据流向图

```
选方 + 背包 + 炮制                   煎药参数数据
     │                                  │
     │ prescription + herbs             │ decoctionParams
     │                                  │
     ▼                                  ▼
┌─────────────────────────────────────────────┐
│              煎药 (DecoctionScene)           │
│                                             │
│  统一场景布局: 左侧炉灶动画 + 右侧背包        │
│  流程: 选方剂→选药材→开始煎煮→收取成品       │

     │
```

### 4.2 核心交互

| 区域 | 交互方式 | 说明 |
|-----|---------|------|
| 右侧方剂信息 | 显示 | 显示当前方剂名称+组成要求 |
| 右侧背包药材 | 点击 | 点击选中/取消药材 |
| 左侧已选药材 | 显示 | 显示✓表示已选中状态 |
| 火候选择 | 点击选择 | 武火/文火二选一 |
| 开始煎煮 | 点击按钮 | 组成齐全后可用 |
| 煎药炉灶动画 | AI生图占位 | 240×300，蒸汽+气泡动效 |
| 成品区域 | 点击收取 | 煎煮完成后点击进背包 |

### 4.3 AI生图需求

| 素材 | 尺寸 | 描述 |
|-----|-----|------|
| 煎药炉灶动画 | 240×300 | 药罐+炉灶+蒸汽+气泡，可静止或简单动效 |
| 药材卡片 | 60×60 | 各药材独立图标 |

## 五、评分系统

### 5.1 评分维度

| 维度 | 权重 | 说明 |
|-----|-----|------|
| composition | 50% | 药材组成齐全且正确 |
| fire | 30% | 火候选择正确(武火/文火) |
| time | 20% | 煎煮时间掌握准确 |

### 5.2 错误类型

```typescript
type DecoctionErrorType =
  | 'missing'    // 缺少药材
  | 'extra'      // 多余药材
  | 'wrong'      // 错误药材
  | 'fire_wrong' // 火候错误
  | 'time_wrong';// 时间错误
```

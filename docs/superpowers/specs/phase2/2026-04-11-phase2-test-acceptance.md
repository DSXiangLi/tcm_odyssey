**版本**: v1.0
**日期**: 2026-04-11
**状态**: 设计完成，待实施验证

---

## 1. 测试范围

Phase 2 包含AI核心改造，需验证以下功能模块：

| 模块 | 功能点 | 风险等级 |
|-----|-------|---------|
| **问诊环节** | AI病人对话、自由追问、线索追踪 | 高 |
| **病案系统** | 病案库、解锁机制、回顾界面 | 中 |
| **Hermes集成** | 自动启动、API可用性、流式输出 | 高 |
| **病人视觉** | 头像生成、预缓存加载 | 低 |

---

## 2. 测试分层

采用 P0/P1/P2 三级测试策略：

| 层级 | 定义 | 覆盖范围 |
|-----|------|---------|
| **P0 (Smoke)** | 基础可用性验证 | 核心流程是否能走通 |
| **P1 (Functional)** | 功能完整性验证 | 各功能点是否按设计工作 |
| **P2 (Deep)** | 边界/异常/性能验证 | 特殊情况、极端输入、性能指标 |

## 8. AI质量评估（人工验证）

以下测试项无法完全自动化，需要人工评估AI质量：

### 8.1 病人对话自然度评估

| 评估维度 | 评分标准 |
|---------|---------|
| **说话风格一致性** | 农夫说"大夫，您看看"，商人说"先生，请教..."，风格区分明显 |
| **内容合理性** | 病人回答与病案描述一致，不出现矛盾 |
| **表达自然度** | 不像AI生成，更像真实患者描述 |
| **追问响应恰当性** | 对玩家追问给出合理补充信息 |

**评分方式**: 1-5分制，3分以上为合格

### 8.2 线索识别准确性评估

| 评估维度 | 评分标准 |
|---------|---------|
| **必须线索识别率** | 正确识别率 ≥ 90% |
| **语义理解准确性** | "身上干干的" → 正确识别为"无汗" |
| **误判率** | 将无关信息误判为线索的频率 ≤ 10% |

## 9. 测试执行计划

### 9.1 执行顺序

```
1. P0 Smoke测试 (所有模块) → 基础流程验证
2. P1 Functional测试 (按模块顺序) → 功能完整性验证
3. P2 Deep测试 → 边界/异常处理
4. 人工AI质量评估 → AI能力最终验证
```

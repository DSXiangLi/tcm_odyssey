## 一、游戏概述

### 1.1 游戏定位

辨证是中医诊断流程的核心环节（问诊→脉诊→舌诊→辨证→选方），玩家综合问诊症状、脉象、舌象信息，运用中医理论判断证型，是连接诊断与治疗的关键。

### 1.2 核心玩法

| 玩法要素 | 描述 |
|---------|------|
| **学习目标** | 掌握中医辨证思维，建立症状→舌脉→证型的逻辑链 |
| **操作方式** | 选择证型 + 论述分析(可选) |
| **难度设计** | 初级(提示+2选项)→中级(无提示+4选项)→高级(论述评分+全选项) |
| **反馈机制** | 正确显示辨证逻辑，错误显示正确分析+症状舌脉对应关系 |

### 1.3 游戏时长

- 单次辨证: 60-120秒
- 完整流程(含论述): 3-5分钟

## 二、与其他游戏关联

### 2.1 上游依赖

| 来源 | 数据依赖 | 说明 |
|-----|---------|------|
| 问诊 | clueData.symptoms | 症状是辨证核心依据 |
| 脉诊 | pulseData.position/tension | 脉象辅助辨证 |
| 舌诊 | tongueData(全部) | 舌象辅助辨证 |
| 病案 | caseData.syndrome | 正确答案 |

### 2.2 下游输出

| 输出数据 | 目标 | 说明 |
|---------|-----|------|
| syndromeType: string | 选方 | 选方时显示已判断证型 |
| syndromeCorrect: boolean | FlowManager | 记录辨证正确性 |
| analysisText: string | NPC反馈 | 青木先生评述论述内容 |

### 2.3 数据流向图

```
问诊 + 脉诊 + 舌诊                  病案数据
     │                                  │
     │ 所有诊断数据                      │ caseData.syndrome
     │                                  │
     ▼                                  ▼
┌─────────────────────────────────────────────┐
│              辨证 (SyndromeScene)            │
│                                             │
│  输入: 症状 + 脉象 + 舌象                    │
│  输出: syndromeType + analysisText          │
│       + syndromeCorrect                     │
└─────────────────────────────────────────────┘
     │
     │ syndromeData
     ▼
选方(Prescription) → NPC反馈 → 煎药(Decoction)
```

## 三、游戏流程设计

### 3.1 主流程

```
开始辨证
    │
    ▼
显示已收集诊断信息汇总 ───────────────────┐
    │                                    │ 可点击查看
    │ 症状: 恶寒、发热、无汗              │ 详细内容
    │ 脉象: 浮紧                          │
    │ 舌象: 淡红、薄白苔                  │
    │                                    │
    ▼                                    │
选择证型                                 │
    │                                    │
    ├── 初级/中级: 单选证型               │
    │                                    │
    ├── 高级: 输入论述分析                │
    │                                    │
    ▼                                    │
确认辨证                                  │
    │                                     │
    ▼                                     │
验证正确性                                │
    │                                     │
    ├── 正确 ──► 显示辨证逻辑             │
    │                                     │
    ├── 部分正确 ──► 显示部分正确分析     │
    │                                     │
    ├── 错误 ──► 显示正确分析+解释        │
    │                                     │
    ▼                                     │
记录到DiagnosisFlowManager               │
    │                                     │
    ▼                                     │
切换到选方场景                            │
```

### 3.2 论述评分流程(高级难度)

```
输入论述分析
    │
    ▼
AI关键词分析
    │
    ├── 关键症状覆盖检查
    │   ├── 恶寒 ✓ (+10分)
    │   ├── 发热 ✓ (+10分)
    │   ├── 无汗 ✓ (+10分)
    │
    ├── 脉象关联检查
    │   ├── 浮脉说明 ✓ (+15分)
    │   ├── 紧脉说明 ✓ (+15分)
    │
    ├── 舌象关联检查
    │   ├── 舌质分析 ✓ (+10分)
    │   ├── 舌苔分析 ✓ (+10分)
    │
    ├── 证型逻辑检查
    │   ├── 风寒表实推理 ✓ (+20分)
    │
    ▼
生成论述评分 + 反馈建议
```

## 五、数据结构

### 5.1 输入数据

```typescript
interface SyndromeInputData {
  caseId: string;
  correctSyndrome: string;      // 正确证型

  // 上游诊断数据
  clueData: {
    symptoms: string[];
  };
  pulseData: {
    position: string;
    tension: string;
  };
  tongueData: {
    body: string;
    coat: string;
    shape: string;
    moist: string;
  };

  // 证型选项列表
  syndromeOptions: string[];

  playerLevel: 'beginner' | 'intermediate' | 'advanced';
}
```

### 5.2 输出数据

```typescript
interface SyndromeOutputData {
  syndromeType: string;         // 玩家选择证型
  analysisText?: string;        // 论述分析内容
  isCorrect: boolean;
  analysisScore?: {
    symptomCoverage: number;    // 症状关键词覆盖
    pulseAnalysis: number;      // 脉象分析得分
    tongueAnalysis: number;     // 舌象分析得分
    logicScore: number;         // 辨证逻辑得分
  };
  totalScore: number;
  feedback: string;
}
```

## 六、评分系统

### 6.1 评分维度

| 维度 | 权重 | 评分标准 |
|-----|-----|---------|
| 证型正确 | 60% | 正确60分，错误0分 |
| 论述评分(高级) | 40% | 关键词覆盖+逻辑分析 |
| 用时加分 | +10% | <60秒+10分 |

### 6.2 论述评分规则

```typescript
const ANALYSIS_KEYWORDS = {
```

## 1. 设计概述

### 1.1 测试定位

Phase 1.5 AI端到端测试聚焦于 **视觉风格验收**，与Phase 1功能性验收互补。

| Phase | 测试范围 | 判定重点 |
|-------|---------|---------|
| Phase 1 | 功能性验证（布局、移动、碰撞、场景切换） | 硬性精确匹配 |
| **Phase 1.5** | **视觉风格验证（氛围、配色、分层、统一性）** | **AI视觉判断 + 用户感知验证** |

### 1.2 核心验收目标

**Phase 1.5完成标志**:
> 玩家可以真正"走进完整的游戏世界"，感受到田园治愈+中医文化+神秘探索的综合氛围

**验收维度**:
| 维度 | 权重 | 验证方式 |
|------|------|---------|
| **氛围传达** | 30% | AI视觉分析 + 用户感知评分 |
| **分层架构** | 20% | 技术验证 + 视觉效果确认 |
| **风格统一性** | 20% | AI视觉分析 |
| **颜色和谐度** | 15% | 颜色提取 + AI分析 |
| **交互清晰度** | 15% | AI视觉分析 + 用户测试 |

### 1.3 核心设计原则

| 原则 | 描述 |
|------|------|
| **最大自动化** | 尽一切可能使用AI测试，把人工投入降到最低 |
| **人工验收精度** | 验收标准参照人工验收的最高标准 |
| **多维验证** | 截图分析 + 颜色提取 + 状态数据 + 用户感知 |
| **阈值判定** | AI置信度≥80%视为通过，用户评分≥4分/5分视为通过 |

## 6. 用户感知验证

### 6.1 验证流程

```
1. 用户打开游戏（http://localhost:3003）
2. 点击"开始游戏"进入百草镇室外
3. 自由移动和交互3-5分钟
4. 填写感知问卷
```

### 6.2 感知问卷

**必答问题**:

| 问题 | 选项 |
|------|------|
| 1. 走进这个世界，你的整体感受是？ | A.非常舒适 B.比较舒适 C.一般 D.有些不适 E.非常不适 |
| 2. 你是否感受到田园治愈的氛围？ | 1-5分 |
| 3. 你是否注意到中医文化元素？ | A.明显 B.有但不突出 C.没有注意到 D.感觉不相关 |
| 4. 场景中的颜色是否让你感到舒适？ | 1-5分 |
| 5. 建筑和元素的风格是否统一？ | 1-5分 |
| 6. 你是否能识别出可交互的元素？ | A.清晰 B.有些模糊 C.不清楚 |

**开放问题**:
- 你最喜欢这个场景的哪个部分？
- 你觉得哪些地方需要改进？
- 这个场景让你想起了什么游戏？

| 指标 | 通过条件 |

| 整体感受 | A+B选项≥70% |
| 氛围评分 | 平均≥3.5/5 |

## 12. 风险与缓解

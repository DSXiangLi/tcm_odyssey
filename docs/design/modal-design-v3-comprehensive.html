弹窗设计方案 v3 - 完善版

核心设计原则

1. 多层弹窗可见性原则

  • 上层弹窗必须比下层更醒目：更强的边框、更低的透明度、更强的外阴影
  • 边框颜色区分层级：底层用绿色边框(#80a040)，顶层用金棕边框(#c0a080)
  • 背景透明度递减：底层0.85-0.90，顶层1.0（完全不透明，完全不透明才能清晰区分层级）

2. 内容宽度约束原则

  • 文字max-width = 弹窗宽度 - 边框宽度*2 - padding*2
  • 进度条有效宽度 = 容器宽度 - stroke宽度*2 - 内部间隙*2
  • 永远不要让内容超出弹窗边界

3. 弹窗类型与方案对应

  • 标题画面/入口弹窗 → 方案A（渐变玻璃，强装饰感）
  • 游戏内主弹窗 → 方案B（3D立体边框，清晰层次）
  • 确认/警告弹窗（顶层） → 方案C（强边框+低透明度）
  • 对话/提示浮层 → 方案D（悬浮卡片）

计算公式

容器宽度: barWidth (如300px)

边框宽度: strokeWidth = 2px

内部间隙: internalPadding = 2px


边框占用: strokePadding = strokeWidth * 2 = 4px

间隙占用: gapPadding = internalPadding * 2 = 4px


有效宽度: effectiveWidth = barWidth - 8px

填充宽度: fillWidth = Math.floor(effectiveWidth * progress)


填充位置: fillX = strokeWidth + internalPadding

填充高度: fillHeight = barHeight - 8px

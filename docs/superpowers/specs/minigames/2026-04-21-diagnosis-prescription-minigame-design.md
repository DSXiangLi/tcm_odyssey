## 一、游戏概述

### 1.1 游戏定位

辨证选方是中医诊断流程的核心环节（问诊→脉诊→舌诊→辨证选方），玩家综合问诊症状、脉象、舌象信息，先判断证型，再选择对应方剂，体现中医"辨证论治"的整体思维。

> **合并原因**: 辨证与选方交互都比较简单（主要是单选选项），合并后减少场景切换，更好体现中医诊疗的连续性思维。

### 1.2 核心玩法

| 玩法要素 | 描述 |
|---------|------|
| **学习目标** | 掌握中医辨证思维 + 方剂与证型对应关系 |
| **操作方式** | 两步决策：①选择证型 → ②选择方剂 → ③确认 |
| **难度设计** | 初级(提示+2选项)→中级(无提示+4选项)→高级(论述+方剂加减) |
| **反馈机制** | 正确显示辨证逻辑+方剂分析，错误显示正确分析 |

### 1.3 游戏时长

- 单次辨证选方: 90-180秒（合并后时间节省约30秒）
- 完整流程(含论述): 4-6分钟

## 六、评分系统

### 6.1 评分维度

| 维度 | 权重 | 评分标准 |
|-----|-----|---------|
| **证型正确** | 40% | 正确40分，错误0分 |
| **方剂正确** | 40% | 正确40分，错误0分 |
| **论述评分**（高级） | 20% | 关键词覆盖+逻辑分析 |
| **用时加分** | +10% | <90秒+10分，90-180秒+5分 |

### 6.2 评分计算

```typescript
function calculateDiagnosisScore(
  syndromeCorrect: boolean,
  prescriptionCorrect: boolean,
  analysisScore?: AnalysisScore,
  timeSpent: number
): DiagnosisScoreResult {
  let baseScore = 0;

  // 证型评分(40%)
  baseScore += syndromeCorrect ? 40 : 0;

  // 方剂评分(40%)
  baseScore += prescriptionCorrect ? 40 : 0;

  // 论述评分(20%，高级难度)
  if (analysisScore) {
    baseScore += analysisScore.total * 0.2;
  }

  // 用时加分
  if (timeSpent < 90) baseScore += 10;
  else if (timeSpent < 180) baseScore += 5;

  const finalScore = Math.min(100, Math.round(baseScore));

  return {
    syndrome_score: syndromeCorrect ? 40 : 0,
    prescription_score: prescriptionCorrect ? 40 : 0,
    analysis_score: analysisScore?.total || 0,
    time_bonus: timeSpent < 90 ? 10 : (timeSpent < 180 ? 5 : 0),
    total_score: finalScore,
    passed: finalScore >= 60,
    diagnosis_correct: syndromeCorrect && prescriptionCorrect
  };
}
```

## 七、方剂知识系统

### 7.1 一期方剂数据（与证型对应）

| 证型 | 推荐方剂 | 核心区别 |

### 7.2 方剂对比要点

```
证型区分要点:

- 无汗 vs 有汗
- 脉紧 vs 脉缓
- 麜黄汤 vs 桂枝汤

风热表证 vs 风温初起:
- 咽痛为主 vs 咳嗽为主
- 银翘散 vs 桑菊饮
```

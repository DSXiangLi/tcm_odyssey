## 一、开发概述

### 1.1 改造目标

将种植游戏改造为"地块网格+资源卡片"交互模式，实现四气五味归经的视觉化学习。

### 1.2 核心特色

| 特色 | 描述 |
|-----|------|
| 地块网格可视化 | 空闲地块网格(简化版，无归经限制) |
| 四气五味卡片 | 水源/肥料选择卡片(图标+颜色) |
| 匹配度展示 | 实时显示水源/肥料匹配度 |
| 生长动画 | 种子→发芽→生长→成熟可视化 |
| **考题系统** | **收获考教(5种题型，最多3次尝试)** |
| **考题奖励** | **尝试次数→经验值倍率(1.5×/1.0×/0.5×)** |

## 四、Phase 3: 生长动画

### 4.1 种植→生长→成熟动画

**动画阶段**:
```
种植动画:
    │
    │  播种: 种子落入地块(飞入动画)
    │         │
    │         ▼
    │  浇水: 水源倒入地块(流动效果)
    │         │
    │         ▼
    │  施肥: 肥料撒入地块(散落效果)
    │
    ▼
生长等待(ProgressBar):
    │
    │  进度: 0% → 25% → 50% → 75% → 100%
    │
    │  状态变化:
    │  0%:   种子(种子图标)
    │  25%:  发芽(嫩芽图标)
    │  50%:  生长(小株图标)
    │  75%:  成长(中株图标)
    │  100%: 成熟(药材图标)
    │
    ▼
收获考教:
    │
    │  显示考教题目(药材功效/主治)
    │
    │  答对 → 收获入库
    │
    │  答错 → 重试或查看解释
```

## 五、Phase 4: 四气五味匹配逻辑

### 5.1 匹配计算可视化

```typescript
interface MatchVisualization {
  plotMatch: {
    expected: string;           // 种子归经(肺经)
    actual: string;             // 地块归经(肺经A)
    match: boolean;             // 是否匹配
    score: number;              // 得分(30分)
  };

  waterMatch: {
    expectedQi: string;         // 种子需求四气(温)
    actualQi: string;           // 水源四气(温)
    match: boolean;
    score: number;              // 得分(35分)
  };

  fertilizerMatch: {
    expectedFlavor: string;     // 种子需求五味(辛)
    actualFlavor: string;       // 肥料五味(辛)
    match: boolean;
    score: number;              // 得分(35分)
  };

  totalScore: number;           // 总分(0-100)
}
```

### 6.4 考题评分逻辑

```typescript
// src/systems/QuizManager.ts

export class QuizManager {
  // 随机选择考题
  selectRandomQuiz(herbId: string): QuizItem;

  // 判断答案
  checkAnswer(selectedIndex: number): boolean;

  // 计算奖励倍率
  calculateRewardMultiplier(attempts: number): number {
    switch (attempts) {
      case 1: return 1.5;  // 首次答对
      case 2: return 1.0;  // 第2次答对
      case 3: return 0.5;  // 第3次答对
      default: return 0;   // 3次全错
    }
  }
}
```

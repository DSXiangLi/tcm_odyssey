## 1. 设计哲学

### 1.1 核心认知转变

| 传统游戏NPC | AI Agent NPC |
|------------|-------------|
| 被动执行预设逻辑的木偶 | 有自主意识的数字生命 |
| 玩家触发 → NPC响应 | NPC有目标、日程、情感 |
| 静态对话树 | 动态生成教学内容 |
| 任务是数据 | 任务是教学手段 |
| 记忆是变量 | 记忆是演变的认知 |

### 1.2 "有灵魂的NPC"定义

一个真正有灵魂的NPC应该具备：

| 维度 | 定义 | Hermes对应 |
|-----|------|-----------|
| **身份认同** | 我是谁？我的专业？我的性格？ | SOUL.md |
| **记忆系统** | 我对这个学生了解什么？ | USER.md |
| **教学大纲** | 我要教什么？教学计划是什么？ | SYLLABUS.md (Skill) |
| **任务管理** | 学生学到哪了？下一步学什么？ | TASKS.json + Tools |
| **自主行动** | 我能主动发起什么？ | Cron + Tools |

## 4. 教学大纲设计 (SYLLABUS.md)

### 4.1 大纲结构原则

**主线选择**：以**中医内科学（病症）**为主线，方剂为二级目录

**层级结构**：
```
病症（外感病）→ 病类（风寒/风热）→ 证型（表实/表虚/犯卫）→ 方剂（麻黄汤/桂枝汤/银翘散）
```

**设计理念**：
- 玩家学到的是"诊治思路"，而非孤立记忆方剂
- 从病机理解→辨证判断→选方用药，形成完整认知链
- 不同学派（伤寒vs温病）有不同的辨证体系

### 4.2 大纲完整内容

```markdown
# 青木先生教学大纲 - 外感病诊治

## 第一篇：外感病总论

### 1.1 外感病概念
- **定义**：外邪侵袭人体引起的疾病
- **病因**：六淫（风寒暑湿燥火）
- **分类**：伤寒（风寒）vs 温病（风热、湿热等）
- **特点**：起病急、病程短、有表证

### 1.2 辨证体系对比
| 辨证体系 | 适用范围 | 代表著作 | 核心概念 |
|---------|---------|---------|---------|
| **六经辨证** | 伤寒（风寒） | 《伤寒论》 | 太阳→阳明→少阳→太阴→少阴→厥阴 |
| **卫气营血辨证** | 温病（风热） | 《温病条辨》 | 卫分→气分→营分→血分 |
| **三焦辨证** | 温病（湿热） | 《温病条辨》 | 上焦→中焦→下焦 |

### 1.3 学派归属
- **伤寒学派**：张仲景《伤寒论》，辛温解表代表
  - 代表方：麻黄汤、桂枝汤、小青龙汤
  - 辨证特点：重在辨别表实/表虚、太阳/阳明
- **温病学派**：吴鞠通《温病条辨》，辛凉解表代表
  - 代表方：银翘散、桑菊饮
  - 辨证特点：重在辨别卫分/气分、风热/湿热

### 1.4 治疗原则
- **辛温解表**：风寒外感，用麻黄汤、桂枝汤
- **辛凉解表**：风热外感，用银翘散、桑菊饮
- **扶正解表**：体虚外感，用参苏饮、再造散

### 1.5 病机传变规律
- **伤寒传变**：太阳病不解 → 入里 → 阳明病（白虎汤证）→ 少阳病（小柴胡汤证）
- **温病传变**：风热犯卫 → 入气分 → 热入营血

## 第二篇：风寒外感（伤寒）

### 2.1 风寒外感总论
- **病机总纲**：风寒束表，肺卫失宣
- **临床特点**：恶寒重、发热轻、无汗或少汗、头痛身痛
- **治疗总法**：辛温解表，宣肺散寒

### 2.2 证型一：风寒表实证（麻黄汤证）

#### 2.2.1 证型定义
- **病机**：风寒束表，营卫郁滞，肺气不宣
- **核心症状**：
  - 恶寒重、发热轻
  - **无汗**（关键鉴别点）
  - 头痛、身痛、骨节疼痛
  - 气喘（肺气郁闭）
- **舌脉特点**：
  - 舌：舌淡苔薄白
  - 脉：**脉浮紧**（紧脉为关键）

#### 2.2.2 辨证要点
| 要点 | 内容 | 记忆方法 |
|-----|------|---------|
| **核心鉴别** | 无汗 vs 有汗 | 表实无汗，表虚有汗 |
| **脉象关键** | 浮紧 vs 浮缓 | 紧如绞绳，缓如怠行 |
| **恶寒程度** | 恶寒重 > 发热 | 寒邪偏重 |
| **体质要求** | 体实者可用 | 虚人禁用峻汗 |

#### 2.2.3 治法方药
- **治法**：辛温发汗，宣肺平喘
- **方剂**：麻黄汤

#### 2.2.4 麻黄汤详解

##### 【方剂身份】
| 属性 | 内容 |
|-----|------|
| **出处** | 《伤寒论》第35条 |
| **地位** | 发汗解表第一方，太阳病伤寒代表方 |
| **学派** | 伤寒学派 |
| **历史意义** | 仲景四大方之首，开创辛温发汗法 |

##### 【组成配伍】
| 位置 | 药材 | 用量 | 职责 | 配伍意义 |
|-----|------|------|------|---------|
| **君** | 麻黄 | 三两 | 宣肺发汗 | 开腠理，散风寒 |
| **臣** | 桂枝 | 二两 | 温经解肌 | 助麻黄发汗，温通经脉 |
| **佐** | 杏仁 | 七十个 | 降肺气 | 助麻黄平喘，降逆止咳 |
| **使** | 甘草 | 一两 | 调和诸药 | 缓峻烈之性，防过汗伤正 |

##### 【配伍精髓】
- **麻黄+桂枝**：强发汗，开腠理散风寒
- **麻黄+杏仁**：宣肺平喘，一宣一降
- **配伍比例**：麻黄:桂枝=3:2（麻黄为主，桂枝为辅）

##### 【煎服法要点】
| 要点 | 方法 | 原因 |
|-----|------|------|
| **煎法** | 先煎麻黄去沫，后下诸药 | 去沫防烦，后下保药效 |
| **服法** | 温服，覆取微汗 | 助药力发汗 |
| **汗法** | 微汗为度，不可大汗 | 大汗伤正，病必不除 |

##### 【禁忌要点】
| 禁忌类型 | 内容 | 原因 |
|---------|------|------|
| **自汗禁用** | 已经出汗者禁用 | 汗出表虚，非表实证 |
| **体虚慎用** | 气虚、阳虚者慎用 | 峻汗伤正，虚人受不了 |
| **孕妇禁用** | 孕妇绝对禁用 | 发汗伤胎 |
| **心脏病慎用** | 心悸、心脏病慎用 | 麻黄兴奋心脏 |

##### 【类方家族】
| 类方 | 加减 | 主治变化 |
|-----|------|---------|
| 麻黄加术汤 | 加白术 | 风寒湿痹 |
| 麩杏甘石汤 | 去桂枝，加石膏 | 表邪入里化热 |
| 大青龙汤 | 加石膏、生姜、大枣 | 表寒里热 |
| 小青龙汤 | 去杏仁，加半夏、干姜、细辛、五味子 | 表寒里饮 |

##### 【方歌记忆】
```
麻黄汤中用桂枝，杏仁甘草四般施，
发热恶寒头项痛，伤寒无汗服之宜。
```

##### 【鉴别要点】
| 对比 | 麻黄汤 | 桂枝汤 |
|-----|--------|--------|
| **汗出情况** | 无汗（表实） | 有汗（表虚） |
| **脉象特点** | 浮紧 | 浮缓 |
| **疼痛程度** | 身痛明显 | 头痛为主 |
| **体质要求** | 体实可用 | 体虚也能用 |
| **发汗力度** | 峻汗 | 微汗 |
| **治法特点** | 发汗解表 | 解肌发表、调和营卫 |

##### 【现代应用】
- 上呼吸道感染（感冒）
- 急性支气管炎、哮喘
- 肺炎初期
- 风湿性关节炎（表寒湿）

### 2.3 证型二：风寒表虚证（桂枝汤证）

#### 2.3.1 证型定义
- **病机**：风邪袭表，营卫不和
- **核心症状**：
  - 恶风（非恶寒）
  - 发热
  - **汗出**（关键鉴别点）
  - 头痛
- **舌脉特点**：
  - 舌：舌淡苔薄白
  - 脉：**脉浮缓**（缓脉为关键）

#### 2.3.2 辨证要点
| 要点 | 内容 | 记忆方法 |
|-----|------|---------|
| **核心鉴别** | 有汗 vs 无汗 | 表虚有汗，表实无汗 |
| **脉象关键** | 浮缓 vs 浮紧 | 缓如怠行，紧如绞绳 |
| **恶风特点** | 恶风非恶寒 | 风邪为主，寒邪次之 |
| **营卫关系** | 营卫不和 | 卫强营弱，汗出 |

#### 2.3.3 治法方药
- **治法**：解肌发表，调和营卫
- **方剂**：桂枝汤

#### 2.3.4 桂枝汤详解

##### 【方剂身份】
| 属性 | 内容 |
|-----|------|
| **出处** | 《伤寒论》第12条 |
| **地位** | **群方之首**，调和营卫祖方，仲景第一方 |
| **学派** | 伤寒学派 |
| **历史意义** | 开创调和法先河，辨证论治起点 |

##### 【组成配伍】
| 位置 | 药材 | 用量 | 职责 | 配伍意义 |
|-----|------|------|------|---------|
| **君** | 桂枝 | 三两 | 解肌发表 | 散风寒，温通卫气 |
| **臣** | 芍药 | 三两 | 敛阴和营 | 收敛营阴，止汗 |
| **佐** | 生姜 | 三两 | 助桂枝解表 | 散风寒，和胃气 |
| **佐** | 大枣 | 十二枚 | 助芍药和营 | 补营阴，和脾胃 |
| **使** | 甘草 | 二两 | 调和诸药 | 调和营卫，缓急 |

##### 【配伍精髓】
- **桂枝+芍药**：一散一收，调和营卫（核心配伍）
  - 桂枝散风寒，温卫气
  - 芍药敛营阴，止汗出
  - 散收相配，补泻兼施
- **生姜+大枣**：调和脾胃，助营卫生化
- **配伍比例**：桂枝:芍药=1:1（散收平衡）

##### 【煎服法要点】
| 要点 | 方法 | 原因 |
|-----|------|------|
| **煎法** | 以水七升，微火煮取三升 | 微火保药效 |
| **服法** | **啜热稀粥**、温覆取汗 | 助药力发汗，补胃气 |
| **汗法** | 微汗遍身，不可大汗淋漓 | 汗出太多伤正气 |
| **服药次数** | 若一服汗出病愈，停后服 | 病愈即止，不伤正 |

##### 【特殊用法】
- **啜热稀粥的意义**：
  1. 补益胃气，助药力发汗
  2. 热粥温阳，助散风寒
  3. 防止发汗太过伤津液
- **温覆的意义**：保暖助汗出

##### 【禁忌要点】
| 禁忌类型 | 内容 | 原因 |
|---------|------|------|
| **无汗禁用** | 无汗表实禁用 | 表实需发汗，桂枝汤力不足 |
| **热证禁用** | 风热证禁用 | 辛温助热 |
| **酒客慎用** | 喜酒者慎用 | 酒热内蕴，桂枝辛温助热 |

##### 【类方家族】
桂枝汤是"加减方之祖"，衍生方剂最多：

| 类方 | 加减 | 主治变化 |
|-----|------|---------|
| 桂枝加葛根汤 | 加葛根 | 太阳病项背强几几 |
| 桂枝加厚朴杏子汤 | 加厚朴、杏仁 | 太阳病兼喘 |
| 桂枝加附子汤 | 加附子 | 太阳病汗出过多、恶风 |
| 桂枝去芍药汤 | 去芍药 | 太阳病胸满 |
| 桂枝新加汤 | 加芍药、生姜、人参 | 太阳病汗后身痛 |
| 桂枝加龙骨牡蛎汤 | 加龙骨、牡蛎 | 遗精、惊悸 |
| 小建中汤 | 加饴糖倍芍药 | 虚劳里急 |
| ... | ... | 20+首衍生方 |

##### 【方歌记忆】
```
桂枝汤治太阳风，芍药甘草姜枣同，
解肌发表调营卫，表虚有汗此方功。
```

##### 【鉴别要点】
| 对比 | 桂枝汤 | 麻黄汤 |
|-----|--------|--------|
| **汗出情况** | 有汗（表虚） | 无汗（表实） |
| **脉象特点** | 浮缓 | 浮紧 |
| **疼痛程度** | 头痛为主 | 身痛明显 |
| **体质要求** | 体虚也能用 | 体实可用 |
| **发汗力度** | 微汗 | 峻汗 |
| **治法特点** | 调和营卫 | 发汗解表 |

##### 【现代应用】
- 上呼吸道感染（感冒）
- 过敏性鼻炎
- 荨麻疹
- 更年期综合征（营卫不和）
- 自汗、盗汗
- 神经衰弱

## 第三篇：风热外感（温病）

### 3.1 风热外感总论
- **病机总纲**：风热袭表，卫气被郁
- **临床特点**：发热重、恶寒轻、咽痛、咳嗽
- **治疗总法**：辛凉解表，清热解毒

### 3.2 证型三：风热犯卫证（银翘散证）

#### 3.2.1 证型定义
- **病机**：风热袭表，卫气被郁，肺失宣降
- **核心症状**：
  - **发热重**、恶寒轻
  - **咽痛**（关键症状）
  - 咳嗽
  - 口渴
- **舌脉特点**：
  - 舌：舌边尖红，苔薄黄
  - 脉：**脉浮数**

#### 3.2.2 辨证要点
| 要点 | 内容 | 记忆方法 |
|-----|------|---------|
| **核心鉴别** | 发热重 > 恶寒 | 热邪为主，风邪次之 |
| **咽痛特点** | 咽痛明显 | 风热上攻咽喉 |
| **舌象** | 舌边尖红 | 热在卫分 |
| **脉象** | 浉数 | 数脉主热 |

#### 3.2.3 治法方药
- **治法**：辛凉透表，清热解毒
- **方剂**：银翘散（主）、桑菊饮（辅）

#### 3.2.4 银翘散详解

##### 【方剂身份】
| 属性 | 内容 |
|-----|------|
| **出处** | 《温病条辨·上焦篇》 |
| **地位** | **辛凉解表第一方**，温病学代表方 |
| **学派** | 温病学派 |
| **历史意义** | 吴鞠通创立，温病学成熟标志，与伤寒学派辛温法对立 |

##### 【组成配伍】
| 位置 | 药材 | 用量 | 职责 | 配伍意义 |
|-----|------|------|------|---------|
| **君** | 金银花 | 一两 | 清热解毒 | 散上焦风热 |
| **君** | 连翘 | 一两 | 清热解毒 | 透邪外出 |
| **臣** | 薄荷 | 六钱 | 疏散风热 | 透邪外出 |
| **臣** | 荆芥 | 四钱 | 疏风解表 | 助透邪 |
| **臣** | 豆豉 | 五钱 | 解表除烦 | 助透邪 |

| **佐** | 桔梗 | 六钱 | 宣肺利咽 | 治咽痛咳嗽 |

##### 【配伍精髓】
- **金银花+连翘**：清热解毒为君，不同于伤寒学派以辛温为君
- **薄荷+荆芥+豆豉**：透邪外出为臣，芳香透散
- **牛蒡子+桔梗**：解毒利咽，治咽痛咳嗽
- **配伍特点**：清热解毒为主，透邪外出为辅

##### 【煎服法要点】
| 要点 | 方法 | 原因 |
|-----|------|------|
| **煎法** | **鲜芦根煎汤**，香气大出即服 | 保留挥发油，勿过煮 |
| **服法** | 病重者二时一服，日三服夜一服 | 持续药力 |
| **注意事项** | 勿过煮，过煮则味厚气薄 | 芳香药物挥发油易散失 |

##### 【特殊用法】
- **勿过煮的意义**：
  - 薄荷、荆芥含挥发油
  - 过煮挥发油散失，透邪力减弱
  - "香气大出即服"保留药效
```

```typescript
function generateCase(syndromeType: string): CaseRecord {
  // 1. 基于证型模板
  const template = getSyndromeTemplate(syndromeType);

  // 2. 随机组合症状（在证型范围内）
  const symptoms = randomSelect(template.possible_symptoms, {
    core: template.core_symptoms,      // 核心症状必选
    optional: randomCount(1, 3)        // 可选症状随机选1-3个
  });

  // 3. 生成病人信息
  const patient = generateRandomPatient();

  // 4. 组装问诊问答
  const inquiry = generateInquiry(symptoms, template);

  // 5. 生成脉诊（基于证型）
  const pulse = generatePulse(template.pulse_pattern);

  // 6. 生成舌诊（基于证型）
  const tongue = generateTongue(template.tongue_pattern);

  return { case_type: "generated", ... };
}
```

#### 7.9.2 总分计算

```typescript
function calculateScore(gameResult: GameResult): number {
  const weights = {
    inquiry: 0.15,
    pulse: 0.15,
    tongue: 0.10,
    syndrome: 0.30,
    prescription: 0.20,
    overall: 0.10
  };

  return (
    gameResult.inquiry_score * weights.inquiry +
    gameResult.pulse_score * weights.pulse +
    gameResult.tongue_score * weights.tongue +
    gameResult.syndrome_score * weights.syndrome +
    gameResult.prescription_score * weights.prescription +
    gameResult.overall_score * weights.overall
  );
}
```

### 7.10 NPC点评系统

#### 7.10.1 点评生成逻辑

```typescript
function generateNpcFeedback(gameResult: GameResult): string {
  const { score, syndrome_correct, prescription_correct, weaknesses } = gameResult;

  let feedback = "";

  // 1. 总体评价
  if (score >= 90) {
    feedback += "诊治思路非常清晰，辨证准确。";
  } else if (score >= 70) {
    feedback += "总体不错，但还有改进空间。";
  } else {
    feedback += "这次诊治有些问题，我们一起来看看。";
  }

  // 2. 具体环节点评
  if (!gameResult.pulse_correct) {
    feedback += "脉诊部分，你对古文的理解还需要加强。";
  }
  if (!gameResult.syndrome_correct) {
    feedback += "辨证环节，{syndrome_type}的关键点是{key_feature}。";
  }
  if (!gameResult.prescription_correct) {
    feedback += "选方方面，{correct_prescription}是{syndrome_type}的首选方。";
  }

  // 3. 薄弱点提醒
  if (weaknesses.length > 0) {
    feedback += `建议复习：${weaknesses.join("、")}。`;
  }

  return feedback;
}
```

### 14.4 游戏结果处理器实现

```typescript
// src/systems/GameResultHandler.ts

export class GameResultHandler {
  private hermesApiUrl: string = 'http://localhost:8642';

  /**
   * 发送游戏结果到 Hermes
   */
  async submitResult(result: GameResult): Promise<GameResultResponse> {
    try {
      const response = await fetch(`${this.hermesApiUrl}/v1/game/result`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(result)
      });

      return await response.json();
    } catch (error) {
      console.error('Failed to submit game result:', error);
      throw error;
    }
  }

  /**
   * 计算进度增量
   */
  private calculateProgressDelta(result: GameResult): number {
    // 根据游戏类型和得分计算进度
    const baseProgress = 0.1; // 每次游戏基础进度
    const scoreBonus = result.scores.total / 1000; // 得分加成
    return Math.min(baseProgress + scoreBonus, 0.3);
  }

  /**
   * 分析薄弱点
   */
  private analyzeWeaknesses(result: GameResult): string[] {
    const weaknesses: string[] = [];

    // 检查各维度得分，低于70分标记为薄弱点
    for (const [dimension, score] of Object.entries(result.scores.breakdown)) {
      if (score < 70) {
        weaknesses.push(dimension);
      }
    }

    return weaknesses;
  }
}
```

```typescript
    const task = await this.getTask(taskId);

    const gameParams = { taskId, todoId, npcId: task.npcId };

    if (taskType === "prescription" && ["composition", "compatibility"].includes(todoId)) {
```

### 18.5 任务完成与解锁

#### 18.5.1 任务完成触发流程

```typescript
class TaskCompletionHandler {
  async handleGameEnd(gameResult: GameResult) {
    // 1. 更新Todo掌握度
    await this.updateTodoMastery(gameResult.taskId, gameResult.todoId, gameResult.score / 100);

    // 2. 检查是否所有Todo完成
    const task = await this.getTask(gameResult.taskId);
    const allCompleted = task.todos.every(t => t.mastery >= 0.7);

    // 3. 全部完成 → 解锁下一个任务 → NPC通知
    if (allCompleted) {
      await this.completeTask(gameResult.taskId);
      const nextTask = await this.unlockNextTask(gameResult.taskId);
      if (nextTask) {
        await this.notifyTaskUnlock(nextTask);
      }
    }
  }
}
```

### 20.5 UI组件实现

#### 20.5.1 进度条组件

```typescript
class ProgressBar extends Phaser.GameObjects.Container {
  private background: Phaser.GameObjects.Rectangle;
  private fill: Phaser.GameObjects.Rectangle;
  private text: Phaser.GameObjects.Text;

  constructor(scene: Phaser.Scene, x: number, y: number, width: number, height: number) {
    super(scene, x, y);

    // 背景条
    this.background = new Phaser.GameObjects.Rectangle(scene, 0, 0, width, height, 0x444444);
    this.add(this.background);

    // 填充条
    this.fill = new Phaser.GameObjects.Rectangle(scene, 0, 0, 0, height, 0x4CAF50);
    this.add(this.fill);

    // 文字
    this.text = new Phaser.GameObjects.Text(scene, width / 2, height / 2, '0%', {
      fontSize: '12px',
      color: '#ffffff'
    }).setOrigin(0.5);
    this.add(this.text);
  }

  setProgress(percent: number) {
    this.fill.width = this.background.width * percent;
    this.text.setText(`${Math.round(percent * 100)}%`);

    // 根据进度改变颜色
    if (percent >= 0.7) {
      this.fill.setFillStyle(0x4CAF50); // 绿色 - 完成
    } else if (percent >= 0.5) {
      this.fill.setFillStyle(0xFFC107); // 黄色 - 进行中
    } else {
      this.fill.setFillStyle(0xF44336); // 红色 - 需要加强
    }
  }
}
```

```typescript
class TodoStatusIcon extends Phaser.GameObjects.Container {
  constructor(scene: Phaser.Scene, todo: Todo) {
    super(scene, 0, 0);

    // 根据状态选择图标
    const iconKey = todo.status === 'completed' ? 'check' :
                    todo.status === 'in_progress' ? 'current' : 'pending';

    const icon = new Phaser.GameObjects.Image(scene, 0, 0, iconKey);
    this.add(icon);

    // Todo名称
    const nameText = new Phaser.GameObjects.Text(scene, 20, 0, todo.name, {
      fontSize: '14px'
    });
    this.add(nameText);

    // 如果已完成，显示分数
    if (todo.status === 'completed') {
      const scoreText = new Phaser.GameObjects.Text(scene, 100, 0, `${todo.score}分`, {
        fontSize: '12px',
        color: '#4CAF50'
      });
      this.add(scoreText);
    }
  }
}
```

### 20.6 数据流转

#### 20.6.1 游戏开始时数据注入

```typescript
// 游戏启动
class GameScene extends Phaser.Scene {
  init(data: { taskId: string; todoId: string; npcId: string }) {
    this.currentTaskId = data.taskId;
    this.currentTodoId = data.todoId;
    this.npcId = data.npcId;
  }

  create() {
    // 从Hermes获取Task数据
    this.taskData = await this.fetchTaskData(this.currentTaskId);

    // 显示任务介绍面板
    this.showTaskIntroPanel(this.taskData);

    // 创建任务状态栏
    this.taskStatusBar = new TaskStatusBar(this, this.taskData);
    this.add.existing(this.taskStatusBar);
  }
}
```

```typescript
// 游戏结束
async endGame(score: number, details: ScoreDetails) {
  // 构造游戏结果
  const gameResult = {
    game_type: this.gameType,
    task_id: this.currentTaskId,
    todo_id: this.currentTodoId,
    npc_id: this.npcId,
    score: score,
    score_details: details,
    timestamp: new Date().toISOString()
  };

  // 发送到Hermes
  const response = await this.submitGameResult(gameResult);

  // 显示成绩反馈面板
  this.showResultPanel({
    score: score,
    details: details,
    todoUpdate: response.todo_update,
    taskProgress: response.task_progress,
    npcFeedback: response.npc_feedback
  });
}
```

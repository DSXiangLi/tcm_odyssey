## 一、开发概述

### 1.1 改造目标

将炮制游戏改造为统一场景布局：左侧炮制灶台动画 + 右侧背包交互，简化流程，强化药材变换的视觉体验。

### 1.2 核心特色

| 特色 | 描述 |
|-----|------|
| 左侧动画区 | 炮制灶台动画(AI生图占位240×300) + 药材变换展示 |
| 右侧背包区 | 药材列表 + 炮制方法选择(炙类/炒类等) + 辅料选择 |
| 药材选中交互 | 点击药材显示✓状态，左侧显示当前药材名 |
| 药材变换动画 | 原始药材→闪烁动效→饮片(60×60) |
| 成品收取 | 底部显示炮制成品，点击收取进背包 |

## 二、Phase 1: 统一场景布局

### 2.1 左侧动画区容器

**任务**: 创建480×600的左侧区域，容纳灶台动画和药材变换展示。

**技术实现**:
```
ProcessingScene左侧布局:
├── AnimationContainer (480×600)
│   ├── HearthAnimation (240×300) - AI生图占位符
│   │   └── 位置: 居中偏上
│   │   └── 内容: 大铁锅+炉灶+火焰
│   │
│   ├── CurrentHerbLabel (480×40)
│   │   └── 位置: 炉灶动画下方
│   │   └── 内容: "当前药材: 甘草"
│   │
│   ├── HerbTransformationDisplay (480×120)
│   │   └── 位置: 药材名下方
│   │   └── 内容: 原始药材 → 动效 → 饮片
│   │   └── 尺寸: 各60×60
│   │   └── 动效: 简单闪烁动画(代码生成)
```

### 2.2 右侧背包区容器

**任务**: 创建320×600的右侧区域，包含药材列表、方法选择、辅料选择。

**技术实现**:
```

├── BackpackContainer (320×600)

│   │   └── 待炮制药材网格(使用ItemSlotComponent)
```

## 三、Phase 2: 核心交互实现

### 3.1 药材选择交互

**交互流程**:
```
点击右侧药材格子:
    │
    ├── 药材未选中
    │   │
    │   ▼
    │   格子显示✓标记
    │   左侧显示"当前药材: XX"
    │   根据药材推荐炮制方法
    │   │
    │   ▼
    │   方法+辅料齐全: "开始炮制"按钮可用
    │
    └── 药材已选中
        │
        ▼
        取消选中(✓消失)
        左侧清空"当前药材"
        "开始炮制"按钮禁用
```

### 3.2 方法选择交互

**交互设计**:
```
炮制方法选择(使用SelectionButtonComponent):

炙类(蜜炙推荐甘草):
    ○ 蜜炙 - 润肺止咳
    ○ 酒炙 - 活血通络
    ○ 醋炙 - 入肝止痛
    ○ 盐炙 - 入肾补肾

炒类:
    ○ 清炒 - 增强健脾
    ○ 麸炒 - 缓和药性
    ○ 土炒 - 增强补脾

选中后显示●，自动匹配推荐辅料
```

### 3.3 辅料选择交互

**交互设计**:
```
辅料选择(根据方法自动推荐):

蜜炙 → 蜂蜜(推荐)
酒炙 → 黄酒
醋炙 → 米醋
盐炙 → 盐水

点击辅料格子选中(显示✓)
```

### 3.4 开始炮制流程

**流程设计**:
```
点击[开始炮制]:
    │
    ├── 检查药材+方法+辅料齐全
    │   不齐全 → 提示缺少内容
    │   齐全 → 继续
    │
    ▼
    左侧灶台动画开始(铁锅+火焰)
    │
    ▼
    药材变换动画:
    │   - 原始药材图 → 闪烁动效 → 饮片图
    │   - 时长: 约2秒
    │
    ▼
    进度条显示(使用ProgressBarComponent)
    │   - 炮制进度: 0% → 100%
    │   - 时长: 约3秒(可调整)
    │
    ▼
    炮制完成
    │   - 动画停止
    │   - 底部出现成品卡片(如"蜜炙甘草")
    │   - 显示评分结果
```

## 四、Phase 3: 成品收取与评分

### 4.1 成品区域设计

**位置**: 底部居中，炮制完成后出现

**交互**:
```
成品卡片显示:
    │
    ├── 位置: 底部居中
    ├── 尺寸: 80×80
    ├── 内容: 炮制成品名称(如"蜜炙甘草")
    │         成品图标(AI生图占位)
    │
    └── 点击收取
        │
        ▼
        成品进背包(processedHerb入库)
        显示评分详情
        显示NPC反馈
```

### 4.2 评分系统

**评分维度**:
| 维度 | 权重 | 计算方式 |
|-----|-----|---------|
| 方法 | 40% | 炮制方法选择是否正确 |
| 辅料 | 40% | 辅料选择是否正确 |
| 时间 | 20% | 炮制时间是否在合理范围 |

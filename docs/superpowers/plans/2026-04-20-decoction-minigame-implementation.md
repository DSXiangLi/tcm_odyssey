## 一、开发概述

### 1.1 改造目标

将煎药游戏改造为统一场景布局：左侧煎药炉灶动画 + 右侧背包交互，简化流程，强化视觉体验。

### 1.2 核心特色

| 特色 | 描述 |
|-----|------|
| 左侧动画区 | 煎药炉灶动画(AI生图占位240×300)，显示已选药材 |
| 右侧背包区 | 方剂组成提示 + 背包药材列表 + 火候选择 |
| 药材选中交互 | 点击药材显示✓状态，左侧同步显示 |
| 煎煮动画 | 炉灶蒸汽+气泡动效(代码生成或AI生图) |
| 成品收取 | 底部显示煎好药汤，点击收取进背包 |

## 二、Phase 1: 统一场景布局

### 2.1 左侧动画区容器

**任务**: 创建480×600的左侧区域，容纳炉灶动画和已选药材展示。

**技术实现**:
```
DecoctionScene左侧布局:
├── AnimationContainer (480×600)
│   ├── HearthAnimation (240×300) - AI生图占位符
│   │   └── 位置: 居中偏上
│   │   └── 动效: 蒸汽+气泡(可代码生成)
│   │
│   ├── SelectedHerbsDisplay (480×120)
│   │   └── 位置: 炉灶动画下方
│   │   └── 内容: 已选药材格子(带✓状态)
│   │   └── 交互: 点击取消选中
```

### 2.2 右侧背包区容器

**任务**: 创建320×600的右侧区域，包含方剂提示、药材列表、火候选择。

**技术实现**:
```
DecoctionScene右侧布局:
├── BackpackContainer (320×600)
│   ├── PrescriptionInfo (320×100)
│   │   └── 当前方剂名称
│   │   └── 组成药材列表(提示)
│   │
│   ├── HerbInventory (320×200)
│   │   └── 背包药材网格(使用ItemSlotComponent)
│   │   └── 交互: 点击选中/取消
│   │   └── 数量显示
│   │
│   ├── FireSelection (320×80)
│   │   └── 武火/文火二选一(使用SelectionButtonComponent)
│   │
│   ├── StartButton (320×60)
│   │   └── "开始煎煮"按钮
│   │   └── 组成齐全后可用
```

## 三、Phase 2: 核心交互实现

### 3.1 药材选择交互

**交互流程**:
```
点击右侧药材格子:
    │
    ├── 药材未选中
    │   │
    │   ▼
    │   格子显示✓标记
    │   左侧已选药材区域添加该药材
    │   检查方剂组成是否齐全
    │   │
    │   ▼
    │   齐全: "开始煎煮"按钮可用
    │   不齐全: 按钮禁用，显示缺少药材
    │
    └── 药材已选中
        │
        ▼
        取消选中(✓消失)
        左侧移除该药材显示
        "开始煎煮"按钮禁用
```

### 3.2 火候选择交互

**交互设计**:
```
火候选择(使用SelectionButtonComponent):
    │
    ├── ○ 武火 - 高温急煎
    │   选中后显示●
    │   评分权重: 30%
    │
    └── ○ 文火 - 温和慢煎
        选中后显示●
        评分权重: 30%
```

### 3.3 开始煎煮流程

**流程设计**:
```
点击[开始煎煮]:
    │
    ├── 检查药材齐全
    │   不齐全 → 提示缺少药材
    │   齐全 → 继续
    │
    ▼
    左侧炉灶动画开始(蒸汽+气泡)
    │
    ▼
    进度条显示(使用ProgressBarComponent)
    │   - 煎煮进度: 0% → 100%
    │   - 时长: 约3秒(可调整)
    │
    ▼
    煎煮完成
    │   - 动画停止
    │   - 底部出现"煎好药汤"成品卡片
    │   - 显示评分结果
```

## 四、Phase 3: 成品收取与评分

### 4.1 成品区域设计

**位置**: 底部居中，煎煮完成后出现

**交互**:
```
成品卡片显示:
    │
    ├── 位置: 底部居中
    ├── 尺寸: 80×80
    ├── 内容: "煎好药汤"图标(AI生图占位)
    │
    └── 点击收取
        │
        ▼
        药汤进背包(decoctionResult入库)
        显示评分详情
        显示NPC反馈
```

### 4.2 评分系统

**评分维度**:
| 维度 | 权重 | 计算方式 |
|-----|-----|---------|
| 组成 | 50% | 方剂组成药材是否齐全 |
| 火候 | 30% | 武火/文火选择是否正确 |
| 时间 | 20% | 煎煮时间是否在合理范围 |

## 一、游戏概述

### 1.1 游戏定位

煎药是药园生产流程的核心环节（种植→炮制→煎药），玩家根据方剂要求，完成药材选择、配伍放置、顺序设置、火候控制、煎煮操作，学习中药煎煮规范。

### 1.2 核心玩法

| 玩法要素 | 描述 |
|---------|------|
| **学习目标** | 掌握君臣佐使配伍、先煎后下顺序、武火文火火候 |
| **操作方式** | 5阶段流程：选方→选药→配伍→顺序→火候→煎煮 |
| **难度设计** | 初级(提示配伍)→中级(无提示)→高级(自定义加减) |
| **反馈机制** | 评分维度展示(配伍/组成/顺序/火候/时间)，错误详情 |

### 1.3 游戏时长

- 单次煎药: 3-5分钟
- 完整流程(含煎煮动画): 5-8分钟

## 三、游戏流程设计

### 3.1 主流程(5阶段)

```
开始煎药
    │
    ▼
阶段1: 选择方剂
    │ 显示处方列表
    │ 选择要煎煮的方剂
    │ 显示方剂组成要求
    ▼
阶段2: 选择药材
    │ 显示背包药材列表(使用ItemSlot组件)
    │ 选择组成所需药材
    │ 检查药材齐全
    ▼
阶段3: 配伍放置
    │ 显示君臣佐使槽位(使用CompatibilitySlot组件)
    │ 将药材拖放/点击放入对应槽位
    │ 检查配伍正确性
    ▼
阶段4: 设置顺序
    │ 显示先煎/后下/同煎选项
    │ 为各药材设置煎药顺序
    │ 显示顺序提示(如麻黄先煎去沫)
    ▼
阶段5: 设置火候
    │ 显示武火/文火/缓火选项
    │ 选择煎煮火候
    │ 设置煎煮时间
    ▼
阶段6: 煎煮过程
    │ 显示煎煮动画(火焰+气泡+颜色变化)
    │ 进度条显示(使用ProgressBar组件)
    │ 可中途暂停/检查
    ▼
阶段7: 评分展示
    │ 显示各维度评分
    │ 显示错误详情
    │ 显示反馈建议
    ▼
结束煎药
```

### 3.2 配伍放置流程

```
显示配伍槽位(君臣佐使)
    │
    │ 每个槽位显示:
    │ - 角色标签(君/臣/佐/使)
    │ - 药材槽位(凹陷)
    │ - 顺序标签(先煎/后下)
    │
    ▼
选择药材放入槽位
    │
    ├── 点击药材格子(ItemSlot)
    │   │
    │   ▼
    │   点击目标槽位
    │   │
    │   ▼
    │   药材放入槽位
    │
    ├── 拖拽药材格子
    │   │
    │   ▼
    │   拖到目标槽位释放
    │   │
    │   ▼
    │   药材放入槽位
    │
    ▼
检查配伍完整性
```

## 五、评分系统

### 5.1 评分维度(已实现于decoction-data.ts)

| 维度 | 权重 | 说明 |
|-----|-----|------|
| compatibility | 40% | 君臣佐使配伍正确 |
| composition | 20% | 药材组成齐全 |
| order | 20% | 先煎后下顺序 |
| fire | 10% | 火候选择正确 |
| time | 10% | 时间掌握准确 |

### 5.2 错误类型

```typescript
type HerbErrorType =
  | 'missing'    // 缺少药材
  | 'extra'      // 多余药材
  | 'role_wrong' // 配伍角色错误
  | 'order_wrong'// 煎药顺序错误
  | 'fire_wrong';// 火候错误
```

## 2. 背包系统设计

### 2.1 背包分类

| 背包类型 | 内容 | 来源 |
|---------|------|------|
| **药材背包** | 基础药材、炮制后药材、炮制辅料 | 种植收获、NPC赠送、采集 |
| **种植背包** | 种子、化肥、水源（四气属性） | NPC赠送、商店解锁 |
| **工具背包** | 镰刀、水桶、药罐、切药刀 | 游戏进程解锁 |
| **知识背包** | 方剂卡片、医书 | Task完成后获得 |

### 2.2 药袋设计

药袋作为药材背包内的分类容器，按功效分类：

```
药材背包
├── [解表袋] → 麻黄、桂枝、荆芥、防风、薄荷、桑叶、菊花
├── [清热袋] → 金银花、连翘、石膏
├── [温里袋] → 干姜、附子（二期）
├── [化痰袋] → 半夏（二期）
└── ...随方剂自然扩展
```

**设计原则**：
- 药袋数量不固定，随方剂自然扩展
- 一期至少包含：解表袋、清热袋（基于大纲方剂）

### 2.3 背包UI设计

**Tab切换式**：

```
┌─────────────────────────────────────────────────────────────┐
│ [药材] [种植] [工具] [知识]                                  │
├─────────────────────────────────────────────────────────────┤
│ 药材背包（当前Tab）                                          │
│                                                             │
│ 药袋分类：[解表袋] [清热袋] [温里袋]                         │
│                                                             │
│ ┌────┐ ┌────┐ ┌────┐ ┌────┐                               │
│ │桂枝│ │麻黄│ │芍药│ │甘草│                               │
│ │ 5 │ │ 3 │ │ 2 │ │ 8 │  ← 数量                           │
│ └────┘ └────┘ └────┘ └────┘                               │
│                                                             │
│ [关闭]                                                      │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 背包触发方式

| 触发方式 | 说明 |
|---------|------|
| **UI按钮** | 游戏界面常驻背包图标，点击打开 |
| **快捷键** | 按B键快速打开背包 |
| **场景触发** | 进入种植区 → 提示可用种植背包 |

### 2.5 游戏中的背包使用

| 游戏环节 | 背包展示方式 |
|---------|-------------|
| **煎药游戏** | 内嵌式（边操作边选药材） |
| **炮制游戏** | 内嵌式（边操作边选药材+辅料） |
| **种植游戏** | 内嵌式（边操作边选种子+材料） |
| **诊治游戏选方** | 全屏式（暂停思考，查看知识背包） |

## 3. 存档系统设计

### 3.1 存档触发

| 触发时机 | 说明 |
|---------|------|
| **手动存档** | 玩家点击存档按钮主动保存 |
| **退出时自动存档** | 退出游戏自动保存当前进度 |
| **重要节点自动存档** | Task完成、病案完成时自动保存 |

### 3.2 存档结构

单存档，包含全部数据：

```typescript
interface SaveData {
  // 存档元信息
  save_version: string;
  saved_at: string;

  // 玩家属性
  player: {
    experience: number;          // 经验值
    unlocked_features: string[]; // 已解锁功能
    achievements: string[];      // 已获成就
  };

  // 背包数据
  inventory: {
    herbs: { [herb_id: string]: number };      // 药材持有
    seeds: { [seed_id: string]: number };      // 种子持有
    tools: string[];                            // 已有工具
    knowledge_cards: string[];                  // 方剂卡片
  };

  // Task进度（从Hermes同步）
  tasks: {
    npc_id: string;
    tasks: Task[];              // TASKS.json内容副本
  }[];

  // 病案历史
  case_history: CaseHistoryRecord[];

  // 场景状态
  scene_state: {
    current_scene: string;
    player_position: { x: number, y: number };
  };
}
```

### 3.3 存档流程

```
存档时：
游戏状态 → 收集数据 → 从Hermes读取TASKS.json → 合并入SaveData → 写入save.json

恢复时：
读取save.json → 写回Hermes TASKS.json → 恢复游戏状态 → 恢复背包数据
```

## 4. 经验值系统设计

### 4.1 设计原则

- **无等级系统**：经验值直接解锁内容
- **全游戏覆盖**：多个渠道可获得经验值
- **数值后续细化**：框架先确定，具体数值实现时调整

### 4.2 经验值获取渠道（框架）

| 渠道 | 经验值（待定） | 说明 |
|-----|---------------|------|
| 完成Task | +? | 每个Task完成时 |
| 病案完成得分 | +? | 根据得分计算 |
| 收集辅助线索 | +? | 每条辅助线索 |
| 每日学习打卡 | +? | 每天首次学习 |
| 成就解锁 | +? | 一次性奖励 |

### 4.3 经验值解锁内容（框架）

| 解锁内容 | 经验值阈值（待定） | 说明 |
|---------|-------------------|------|
| 方剂加减功能 | ? | 输入时显示未解锁按钮 |
| 新区域（药王谷） | ? | 解锁探索区域 |
| 新NPC | ? | 解锁更多导师 |

### 4.4 成就系统（框架）

| 成就类型 | 示例 | 奖励 |

## 7. NPC帮助机制

### 7.1 自由对话求助

玩家直接在对话框输入求助问题：

```
玩家："我不太理解这个脉象..."
      ↓
青木先生："紧脉的特点是左右弹指、如绞转索，主寒主痛..."
```

### 7.2 NPC主动观察介入

NPC监测玩家状态，主动给出提示：

| 触发条件 | NPC行为 |
|---------|---------|
| 环节停留超过2分钟 | "如果对某个环节不理解，可以问我..." |
| 连续错误3次 | "看来这个知识点需要复习，要不要我讲解一下？" |
| 必须线索未收集但有提示 | "你还没问出汗的情况，这是关键鉴别点..." |

## 8. 时间触发系统

### 8.1 登录触发替代定时任务

原文档设计的Cron定时任务改为登录/场景触发：

| 原设计 | 新设计 |
|-------|-------|
| 每天8:00晨间问候 | 玩家上线后首次与NPC对话触发 |
| 每天10:00复习提醒 | 每次上线时检查薄弱点，NPC主动提及 |
| 每3小时病人来诊 | 玩家进入诊所时触发"是否有病人等待" |
| 任务完成后解锁通知 | 任务完成时立即弹出解锁提示 |

### 8.2 场景触发流程

```
玩家进入诊所：
      ↓
检查是否有：
  - 进行中的诊治Task？
  - 未解锁的病案？
  - 薄弱点需要复习？
      ↓
青木先生根据状态主动对话：
  - "有位病人正在等待诊治..."
  - "你上次在脉诊上有些薄弱..."
  - "今天来学新内容吧..."
```

## 9. 诊治场景过渡

### 9.1 触发流程

```
NPC主动发布任务：
青木先生（对话/消息）→ "有病人来诊，来诊所实践吧"
→ 玩家前往诊所 → 进入诊所弹出确认 → 玩家确认 → 切换到诊治游戏场景
```

## 10. 放弃任务处理

### 10.1 设计原则

**不允许放弃任务**，理由：
- 这是学习游戏，有教学大纲顺序
- 放弃会打乱学习路径
- NPC是AI，玩家可随时求助

### 10.2 替代方案

如果玩家觉得Task太难：
- 通过自由对话向NPC求助
- NPC会给出提示、讲解、安排复习
- 不提供"放弃"选项

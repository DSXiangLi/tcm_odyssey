核心问题

  • 尺寸不一致: InventoryUI固定60×60, 其他UI动态计算
  • 边框风格不统一: InventoryUI用Neumorphism, 其他用纯文字背景
  • 选中状态表示混乱: ○/●符号 vs Neumorphism凸起 vs 颜色变化
  • 交互反馈不一致: hover效果、选中效果各不相同
  • 无统一基类: 每个UI独立实现createButton/ItemSlot等方法

建议的UI组件层级架构

UI组件层级结构
BaseUIComponent (抽象基类)
├── drawBorder()          # 绘制边框(3D/Neumorphism/玻璃态)
├── createExitButton()    # 标准退出按钮(右上角固定位置)
├── getStandardSize()     # 获取标准尺寸
└── destroy()             # 标准销毁流程

ModalUI (弹窗基类, extends BaseUIComponent)
├── 标准尺寸: 720×480 (诊断类) / 640×420 (问诊类)
├── 边框方案: 方案B(3D立体)为主
├── 退出按钮位置: (340, -250) 固定
└── 内部容器标准化

├── DialogUI              # NPC对话
├── InquiryUI             # 问诊界面
├── PulseUI               # 脉诊
├── TongueUI              # 舌诊
├── SyndromeUI            # 辨证
├── PrescriptionUI        # 选方
├── DecoctionUI           # 煎药
├── ProcessingUI          # 炮制
└── PlantingUI            # 种植

ItemSlotComponent (物品格子组件)
├── 标准尺寸: 60×60
├── 边框方案: 方案4(Neumorphism)
├── 内容布局: 图标+数量角标+选中状态
└── 交互: 点击选择, hover凸起

├── InventoryUI.slots     # 背包物品格子
├── DecoctionUI.herbSlots # 煎药药材格子
├── ProcessingUI.herbSlots # 炮制药材格子
└── PlantingUI.seedSlots  # 种子格子

SelectionButtonComponent (选择按钮组件)
├── 尺寸: 固定宽度(选项文字自适应)
├── 边框方案: 方案4(Neumorphism)或方案8(内凹槽位)
├── 选中状态: ○→● 或 凸起效果
└── 交互: 点击切换, hover高亮

├── PulseUI.options       # 脉位/脉势选择
├── TongueUI.options      # 舌象属性选择
├── SyndromeUI.options    # 证型选择
└── PrescriptionUI.options # 方剂选择

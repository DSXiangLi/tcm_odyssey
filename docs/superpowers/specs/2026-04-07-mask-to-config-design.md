## 2. 失败尝试记录 ⚠️

> **重要**: 以下方案已尝试并失败，避免重复踩坑

### 2.1 ❌ 失败方案: AI分块识别

**方案描述:**
- 将小镇全景图切割成多个小块（如10×8=80个区域）
- 使用多模态AI（QWEN VL）逐块识别元素类型
- 根据AI识别结果生成瓦片配置

**失败原因:**
1. **效率低下**: 每块需要单独调用AI API，80个区域需要80次调用
2. **识别不准确**: AI对小块图片的识别结果不一致，边界判断模糊
3. **融合困难**: AI生成的素材与背景图融合感差，元素风格不统一
4. **成本高昂**: 大量API调用，时间和经济成本都很高

**相关文件:**
- `tile_mapping_v2.py` - v2版本分块识别
- `tile_mapping_v3.py` - v3版本改进尝试
- `tile_mapping_v31.py` - v3.1版本
- `tile_mapping_v32.py` - v3.2版本
- `tile_mapping_v4.py` - v4版本10×8区域划分

**结论:** AI分块识别方案不适用于大规模地图的精确坐标提取。

### 2.2 ❌ 失败方案: Tile-by-Tile逐瓦片识别

**方案描述:**
- 将图片切割成32×32像素的瓦片
- 使用AI识别每个瓦片的类型

**失败原因:**
1. **瓦片数量巨大**: 86×48=4128个瓦片，需要4128次AI调用
2. **单个瓦片信息不足**: 32×32像素太小，AI难以准确判断内容
3. **边界瓦片识别困难**: 混合内容的瓦片无法准确分类

**结论:** 逐瓦片识别方案不可行。

### 2.3 ❌ 失败方案: 让AI直接输出坐标

**方案描述:**
- 给AI看带网格标注的图片
- 让AI直接输出"建筑A在(3,3)位置"

**失败原因:**
1. **多模态模型坐标能力弱**: AI对像素坐标的判断不准确
2. **压缩图片导致坐标偏移**: AI处理时图片会被压缩，坐标不对应原图
3. **数值输出不稳定**: AI给出的坐标数值经常与实际位置偏差较大

**结论:** 多模态模型不适合直接输出精确坐标数值。

### 2.4 失败尝试总结

| 方案 | 问题 | 适用场景 |
|-----|------|---------|
| AI分块识别 | 效率低、识别不准、成本高 | 不适用 |
| 逐瓦片识别 | 瓦片数太多、信息不足 | 不适用 |
| AI直接输出坐标 | 坐标能力弱、偏移大 | 不适用 |

**正确方向**: 让AI负责视觉标注（画遮罩层），自动化脚本负责坐标计算。

## 3. 新方案: 遮罩层像素分析

### 3.1 核心思路

**「AI画遮罩，脚本算坐标」**：
- AI: 生成黑白遮罩层（黑色=可行走，白色=障碍，特殊颜色标记门/交互点）
- 脚本: 像素分析 → 连通域检测 → 瓦片坐标转换 → 配置输出

### 3.2 处理流程


## 4. 技术细节

### 4.1 尺寸对齐策略

**问题:** 遮罩层(2784)比原图(2752)多32像素

**验证流程:**
1. 生成叠加图：原图 + 遮罩层黑色轮廓（红色线条）
2. 人工检查建筑/道路边界是否对齐
3. 根据对齐情况选择策略：
   - 如果右侧多余 → 裁剪右侧32像素
   - 如果左右都有偏移 → 居中裁剪
   - 如果对齐良好 → 不调整

### 4.2 瓦片可行走判定

```python
for each tile (32×32 pixels):
    black_pixel_ratio = count_black_pixels / 1024

    if black_pixel_ratio > 0.70:
        mark as "candidate_walkable"
    else:
        mark as "blocked"

# 然后进行连通性检查
BFS from center tile
keep only tiles connected to main region
isolated candidate tiles → mark as "blocked"
```

### 4.3 连通验证测试

| 测试项 | 验证方式 | 通过条件 |
|-------|---------|---------|
| 主连通域存在 | BFS从中心瓦片开始 | 找到面积>50瓦片的连通域 |
| 门可达 | BFS从门瓦片出发 | 门瓦片与主连通域相连 |
| 水井可达 | BFS从水井瓦片出发 | 水井瓦片与主连通域相连 |
| 无孤立可行走瓦片 | 检查所有可行走瓦片 | 全部属于主连通域 |

```typescript
      id: 'well',

    noIsolatedTiles: true
  }
```

## 一、游戏愿景

### 1.1 游戏定位

**药灵山谷**是一款2D像素风格的中医学习游戏，核心体验是"平衡融合"：
- **学习有成就感** - 系统化的中医知识教学
- **探索有新奇感** - 药王谷的发现与收集乐趣
- **生活有放松感** - 药膳制作、家园建设的治愈体验

### 1.2 目标玩家

- 对中医感兴趣的普通人
- 希望轻松学习中医知识的学习者
- 喜欢模拟经营类游戏的玩家

### 1.3 核心特色

1. **AI NPC系统** - 每个核心NPC由LLM驱动，有记忆、有个性、能自主对话
2. **游戏化学习** - 在操作中自然记住药物属性，而非死记硬背
3. **地道药材** - 药王谷地貌对应中国真实产区
4. **无压力节奏** - 无时间/体力限制，玩家自由决定节奏

## 二、核心设计理念

### 2.1 游戏化学习

**核心理念**：让学习过程本身变得有趣，而不是"先学后玩"

玩家在游戏中完成学习，而不是为了玩游戏被迫学习。

### 2.2 操作中记忆

通过游戏操作自然记住药物属性：

| 药物属性 | 游戏机制 | 记忆方式 |
|----------|----------|----------|
| 产地 | 药王谷地貌区域 | 不同地貌对应不同产区 |
| 四气（寒热温凉） | 水源选择 | 用对应性质的水浇灌 |
| 五味（酸苦甘辛咸） | 肥料选择 | 用对应味道的肥料施用 |
| 归经 | 药田分区 | 在对应经络区域种植 |
| 功效/主治 | 药袋分类 | 成熟后考教，放入对应药袋 |

### 2.3 简化原则

- NPC只和玩家对话，NPC之间不交互
- 统一对话接口（任务模式 + 对话模式）
- 无时间/体力限制

### 4.2 场景切换机制

**技术方案**：瓦片地图 + 门瓦片触发

1. 地图由32x32像素瓦片组成
2. 每个瓦片有属性：可通行/不可通行/门/交互点
3. 玩家站到"门"瓦片上按确认键，触发场景切换
4. 使用Tiled Map Editor编辑地图

**场景切换流程**：
```
室外地图 → 站到门瓦片 → 按确认键 → 加载室内地图 → 玩家出现在入口位置
```

## 五、NPC系统设计

### 5.1 NPC分类

| 类型 | 一期内容 | 特点 |
|------|----------|------|
| **核心NPC** | 青木先生 + 老张 | AI驱动、有记忆、有任务线、承担教学 |
| **功能NPC** | 框架预留 | 模板对话、交易功能 |

### 5.2 双核心NPC设定

**青木先生（临床导师）**：
- 性格：温和严谨，善于用比喻讲解复杂概念
- 一期教学：解表类方剂（桂枝汤、麻黄汤、银翘散、桑菊饮）
- 任务类型：观察诊治 → 辅助诊治 → 独立诊治

**老张（药物导师）**：
- 性格：朴实憨厚，说话直来直去，对药材有深厚感情
- 一期教学：基础药材（甘草、薄荷、金银花、菊花、枸杞）+ 基础炮制
- 任务类型：药材考教 → 种植任务 → 炮制任务

### 5.3 TASK + MEMORY + LLM 架构

**简化原则**：NPC只和玩家对话，NPC之间不交互

```
┌─────────────────────────────────────────────────┐
│                   玩家                           │
└─────────────────────┬───────────────────────────┘
                      │
                      ▼ 统一对话接口
┌─────────────────────────────────────────────────┐
│                   NPC                            │
├─────────────────────────────────────────────────┤
│  📋 TASK文件        │  📄 MEMORY文件             │
│  - 任务链定义        │  - 玩家交互记忆            │
│  - 任务状态管理      │  - 教学进度                │
│  - 触发条件         │  - 常见错误                │
├─────────────────────────────────────────────────┤
│                   🤖 LLM                         │
│          基于TASK+MEMORY生成对话                  │
└─────────────────────────────────────────────────┘
```

**数据文件结构**：
```
npcs/
├── qingmu/
│   ├── profile.json   # 角色设定（静态）
│   ├── memory.json    # 交互记忆（动态）
│   └── tasks.json     # 任务链定义
├── laozhang/
│   ├── profile.json
│   ├── memory.json
│   └── tasks.json
└── functional/
    └── templates.json # 功能NPC对话模板
```

### 5.4 双模式统一对话

| 模式 | 功能 | 示例 |
|------|------|------|
| **任务模式** | 发布任务、更新状态、判断完成、给予奖励 | 玩家完成诊治 → NPC自动判断 → 发奖励 → 解锁下一任务 |
| **对话模式** | 自由对话、回答问题、闲聊 | 玩家问"桂枝汤和麻黄汤区别？" → NPC详细解释 |

## 七、探索系统设计

### 7.1 药王谷地貌分区

| 地貌 | 模拟地区 | 气候 | 特产药材 |
|------|----------|------|----------|

| 温带森林 | 东北长白山 | 寒冷湿润 | 人参、五味子、细辛 |

| 热带雨林 | 云南、海南 | 高温多雨 | 砂仁、豆蔻、槟榔 |
| 矿山区 | - | - | 朱砂、石膏、滑石、龙骨 |

### 7.2 采药流程

```
发现药材 → 挖掘采集 → 辨认药材 → 收获入库
```

**发现机制**：
- 在对应地貌区域随机出现（高频率，保持成就感）
- 季节影响：春季花叶类多，秋季根茎类多
- 天气影响：雨后菌类多，晴天花果类多

**挖掘小游戏**：
- 点击挖掘，找到药材根部
- 后期可加入工具升级（小铲子→锄头→专业工具）

**辨认系统**：
- 展示药材真实图片
- 提供特征描述
- 玩家选择正确名称才能收获

## 九、一期MVP方案

### 9.1 一期范围

**包含内容**：
- ✅ 场景系统：百草镇中心区 + 3个室内场景 + 药王谷3-4个区域
- ✅ NPC系统：青木先生 + 老张（AI对话）
- ✅ 学习系统：4个方剂 + 5种药材 + 种植/炮制/药袋
- ✅ 探索系统：药王谷地貌探索 + 采集/挖掘/捕捉
- ✅ 生活系统：玩家之家 + 药膳制作
- ✅ 玩家系统：角色移动、背包、无时间限制

**不包含**：
- ❌ 功能NPC交易系统
- ❌ 钓鱼、家园装饰
- ❌ 节日活动
- ❌ 经济系统（金币）

### 9.2 一期学习内容

**方剂（青木先生教学）**：
1. 桂枝汤 - 风寒表虚证
2. 麻黄汤 - 风寒表实证
3. 银翘散 - 风热表证
4. 桑菊饮 - 风温初起

**药材（老张教学）**：
1. 甘草 - 补气、调和诸药
2. 薄荷 - 疏散风热
3. 金银花 - 清热解毒
4. 菊花 - 疏风清热
5. 枸杞 - 滋补肝肾

### 9.3 一期开发优先级

1. **P0 - 核心框架**：瓦片地图系统、场景切换、玩家移动
2. **P1 - NPC系统**：AI对话接口、TASK/MEMORY文件管理
3. **P2 - 学习系统**：种植、炮制、药袋、考教

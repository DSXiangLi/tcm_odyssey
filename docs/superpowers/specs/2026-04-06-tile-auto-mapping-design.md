## 1. 背景与问题

### 1.1 当前困境

Phase 1.5的素材生成遇到了核心问题：
- AI生成的全景图中，建筑位置是随机的
- 与代码中的碰撞检测、门位置完全对不上
- 之前的素材已全部回退

### 1.2 新方案思路

**「视觉优先，逻辑后推导」**：
1. 先让AI生成完整小镇全景图（美观优先）
2. 在图片上叠加网格+坐标标注
3. 用AI视觉识别各元素的精确坐标
4. 生成瓦片配置映射到游戏代码

## 3. 自动化工作流设计

### 3.1 完整流程

```
┌─────────────────────────────────────────────────────────────┐
│ 输入: 小镇全景图 (用户已生成 version3)                        │
│       tests/visual/asset-test/ai-generated/town_new/         │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 1: 网格叠加                                             │
│ • 缩放图片至 1280×960 (40×30瓦片)                           │
│ • 叠加网格线 + 坐标标注 (0,0) 到 (39,29)                     │
│ • 输出: 带网格的图片                                         │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 2: AI初步识别                                           │
│ • 识别街道坐标范围                                           │
│ • 识别建筑坐标范围                                           │
│ • 识别水域/桥梁坐标                                          │
│ • 识别门位置                                                 │
│ • 输出: 初步瓦片配置 JSON                                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 3: 颜色蒙版生成                                         │
│ • 🟢 绿色 = 可行走区域 (草地、道路)                          │
│ • 🔴 红色 = 障碍区域 (建筑、水域)                            │
│ • 🟡 黄色 = 门 (可交互入口)                                  │
│ • 🔵 蓝色 = 桥梁                                            │
│ • 输出: 颜色蒙版图                                           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 4: AI验证复核                                           │
│ 检查项:                                                      │
│ □ 道路是否连通？（无孤立区域）                               │
│ □ 门是否在建筑边缘？                                         │
│ □ 水域是否完整包围？                                         │
│ □ 有无遗漏障碍物？                                           │
│ □ 坐标是否都在有效范围？                                     │
│ 输出: 检查结果 + 修正建议                                    │
└─────────────────────────────────────────────────────────────┘
                              │
                    ┌─────────┴─────────┐
                    │                   │
              未通过 ▼             通过 ▼
┌───────────────────────┐   ┌───────────────────────────────┐
│ Step 5: 自动修正        │   │ Step 6: 输出最终配置          │
│ • 根据AI建议修正配置    │   │ • 生成 tile-config.json       │
│ • 迭代至Step 4         │   │ • 生成颜色蒙版图              │
│ • 最多5次迭代          │   │ • 输出映射报告                │
└───────────────────────┘   └───────────────────────────────┘
```

### 3.2 颜色编码标准

| 颜色 | RGB | 含义 | 瓦片类型ID | 是否可行走 |
|-----|-----|------|-----------|-----------|
| 🟢 绿色 | (0, 200, 0) | 可行走区域 | 0=草地, 1=道路 | ✅ 是 |
| 🔴 红色 | (200, 0, 0) | 障碍区域 | 2=墙, 3=建筑 | ❌ 否 |
| 🟡 黄色 | (255, 200, 0) | 门 | 4=门 | ✅ 是 |
| 🔵 蓝色 | (0, 100, 200) | 桥梁 | 7=桥 | ✅ 是 |
| ⬜ 白色 | (255, 255, 255) | 未识别 | - | 待确认 |

### 3.3 AI验证检查项

| 检查项 | 通过条件 | 失败处理 |
|-------|---------|---------|
| 道路连通性 | 从任意可行走点能到达所有可行走点 | 标记孤立区域为障碍或添加连接 |
| 门位置有效性 | 门在建筑边缘，且相邻可行走区域 | 调整门坐标到边缘 |
| 水域完整性 | 水域区域完整包围，无缺口 | 补充水域瓦片 |
| 障碍物遗漏 | 无未标记的障碍物 | 颜色采样二次确认 |
| 坐标范围 | 所有坐标在(0,0)到(39,29)内 | 裁剪越界坐标 |

### 3.4 迭代终止条件

- ✅ 所有检查项通过 → 输出最终配置
- ⚠️ 迭代达到5次 → 输出当前最佳配置 + 警告
- ❌ 无法自动修正 → 请求人工干预

## 4. 核心区域定义

### 4.1 必须精确匹配的坐标（Phase 1已有）

这3个建筑的门坐标必须与 `map-config.ts` 精确匹配：

| 建筑 | 起始瓦片 | 尺寸 | 门坐标 | 对应场景 |
|-----|---------|-----|--------|---------|
| 青木诊所 | (3, 3) | 8×6 | (7, 8) | ClinicScene |
| 老张药园 | (29, 3) | 8×6 | (33, 8) | GardenScene |
| 玩家之家 | (3, 21) | 6×6 | (6, 26) | HomeScene |

### 4.2 识别优先级

1. **最高优先级**: 3个核心建筑的门位置
   - 必须精确匹配代码中的坐标
   - 如果AI生成的图片中建筑位置不同，需要：
     - 方案A: 调整代码中的坐标
     - 方案B: 重新生成图片

2. **高优先级**: 道路网络
   - 必须连通所有核心建筑门
   - 宽度建议2-3瓦片

3. **中优先级**: 水域和桥梁
   - 桥梁必须连接两岸道路
   - 水域必须完整包围

4. **低优先级**: 其他建筑和装饰
   - 标记为障碍物即可
   - 后续Phase再添加交互

## 5. 输出物

### 5.1 必须产出

| 输出文件 | 格式 | 用途 |
|---------|-----|------|
| `tile-config.json` | JSON | 瓦片类型配置，供游戏代码使用 |
| `color_mask.png` | PNG | 颜色蒙版图，可视化验证 |
| `mapping_report.md` | Markdown | 识别结果报告 |

### 5.2 tile-config.json 结构

```json
{
  "version": "1.0",
  "source_image": "new_town_nanobanana3.jpeg",
  "grid_size": [40, 30],
  "tile_size": 32,

  "tilemap": [
    [0, 0, 0, 2, 2, 2, 2, 0, 0, ...],  // y=0
    [0, 0, 0, 2, 2, 2, 2, 0, 0, ...],  // y=1
    // ... 30行
  ],

  "buildings": [
    {
      "name": "青木诊所",
      "start": [3, 3],
      "size": [8, 6],
      "door": [7, 8],
      "scene": "ClinicScene"
    },
    // ...
  ],

  "doors": [
    { "x": 7, "y": 8, "building": "青木诊所" },
    { "x": 33, "y": 8, "building": "老张药园" },
    { "x": 6, "y": 26, "building": "玩家之家" }
  ],

  "walkable_types": [0, 1, 4, 7],
  // 0=草地, 1=道路, 4=门, 7=桥梁

  "validation": {
    "roads_connected": true,
    "doors_valid": true,
    "water_complete": true,
    "all_coords_in_range": true
  }
}
```

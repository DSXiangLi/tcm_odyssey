## 核心问题诊断

### 问题1: 色系定义差异

AI评估使用的"场景提取色系"与当前`ui-color-theme.ts`存在差异：

| AI建议色系 | 当前常量 | 差距 |
|-----------|---------|------|
| 绿系主色 `#4a7c59` | `#80a040` | AI建议更柔和、低饱和度 |
| 绿系辅色 `#6b8e4e` | `#60a040` | 需新增辅色定义 |
| 灰蓝系 `#5c6b7a` | `#408080` | AI建议更温暖 |
| 土黄系 `#c4a35a` | 无定义 | **需新增** |
| 暖米黄 `#f5e6d3` | 无定义 | **需新增**（次要文字色） |
| 棕系主色 `#8b6f47` | `#604020` | AI建议更暖棕 |
| 棕系暗色 `#6b5b3d` | 无定义 | **需新增** |

**根本原因**: 当前ui-color-theme.ts从场景PNG直接提取像素颜色，但这些颜色可能过于饱和/鲜艳，不符合"田园治愈"氛围所需的柔和感。

### 问题2: 面板背景色不统一

低分场景（NPC-02, DIAG-04等）评估发现：

| 场景 | 发现问题 | 评估得分 |
|-----|---------|---------|
| NPC-02 | 面板背景为纯黑色，边框为绿色而非棕系 | 40分 |
| DIAG-04 | 面板背景为深青绿色#2c6b6a，不符合灰蓝/土黄系 | 50分 |
| SCENE-03 | 顶部标题栏使用荧光绿#00ff00残留 | 65分 |

**说明**: 这可能是截图采集时的特殊情况，需要实际检查UI代码确认。

### 问题3: 次要文字颜色规范缺失

AI评估反复建议次要文字使用`#f5e6d3`（暖米黄），但当前常量文件未定义：
- `TEXT_SECONDARY` 应为 `#f5e6d3` 而非当前的 `#c0a080`
- 需确保次要文字在所有UI中统一使用暖米黄

## 优化方案

### 方案A: 扩展颜色常量定义

新增以下颜色常量到`ui-color-theme.ts`：

```typescript
// 新增色系（AI建议的柔和色调）
export const UI_COLORS = {
  // ...保持现有定义

  // 新增柔和色系
  GREEN_SOFT: 0x4a7c59,      // 绿系主色（柔和版）
  GREEN_AUXILIARY: 0x6b8e4e, // 绿系辅色
  BLUE_WARM: 0x5c6b7a,       // 灰蓝系（温暖版）
  YELLOW_EARTH: 0xc4a35a,    // 土黄系（新增）
  YELLOW_WARM: 0xf5e6d3,     // 暖米黄（次要文字专用）
  BROWN_MAIN: 0x8b6f47,      // 棕系主色（温暖版）
  BROWN_DARK: 0x6b5b3d,      // 棕系暗色（新增）
};

export const UI_COLOR_STRINGS = {
  // ...保持现有定义

  // 新增字符串常量
  GREEN_SOFT: '#4a7c59',
  GREEN_AUXILIARY: '#6b8e4e',
  BLUE_WARM: '#5c6b7a',
  YELLOW_EARTH: '#c4a35a',
  YELLOW_WARM: '#f5e6d3',  // TEXT_SECONDARY应替换为此
  BROWN_MAIN: '#8b6f47',
  BROWN_DARK: '#6b5b3d',
};
```

### 方案B: 调整现有UI组件颜色

| 组件类型 | 当前颜色 | 调整建议 |
|---------|---------|---------|
| **面板背景** | PANEL_PRIMARY | 低分场景可能需要调整为BLUE_WARM或YELLOW_EARTH |
| **次要文字** | TEXT_SECONDARY (#c0a080) | 替换为YELLOW_WARM (#f5e6d3) |
| **边框装饰** | BORDER_PRIMARY | 部分场景调整为BROWN_MAIN/BROWN_DARK |
| **选中高亮** | BUTTON_PRIMARY (#80a040) | 可考虑调整为GREEN_SOFT (#4a7c59) |

### 方案C: NPC场景专项修复

**问题根源**: NPC对话UI可能使用了非统一的背景色

**修复任务**:
1. 检查DialogUI.ts背景创建代码
2. 确认是否使用PANEL_PRIMARY而非硬编码黑色
3. 添加半透明效果（alpha保持0.85）
4. 检查NPCInteraction.ts中的NPC状态显示UI

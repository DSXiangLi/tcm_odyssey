## 一、开发概述

### 1.1 改造目标

将辨证游戏从"文字选项"模式改造为"证型卡片选择+诊断信息汇总"模式。

### 1.2 核心特色

| 特色 | 描述 |
|-----|------|
| 诊断汇总 | 显示问诊+脉诊+舌诊已收集信息 |
| 证型卡片 | 显示证型特征关键词 |
| 论述评分 | AI关键词分析评分(高级难度) |
| 信息查看 | 点击汇总信息查看详情弹窗 |

## 二、Phase 1: 数据层改造

### 2.1 证型知识数据

**文件**: `src/data/syndrome-knowledge-data.ts`

```typescript
interface SyndromeKnowledgeData {
  syndromeType: string;         // 证型(风寒表实证/风寒表虚证...)

  // 关键特征(用于卡片显示)
  keySymptoms: string[];        // 关键症状(恶寒+发热+无汗)
  keyPulse: string;             // 关键脉象(浮紧)
  keyTongue: string;            // 关键舌象(淡红薄白)

  // 描述信息
  ancientDescription: string;   // 古文描述
  modernDescription: string;    // 现代解释
  clinicalSignificance: string; // 临床意义

  // 关联信息
  relatedPrescriptions: string[]; // 对应方剂
  differentialPoints: string[];   // 与相似证型的区别要点
}
```

## 五、论述评分系统(高级难度)

### 5.1 AI关键词分析

**评分维度**:
| 维度 | 权重 | 关键词检查 |
|-----|-----|-----------|
| 症状覆盖 | 30% | 检查是否提到恶寒/发热/无汗 |
| 脉象关联 | 25% | 检查是否解释浮/紧脉意义 |
| 舌象关联 | 20% | 检查是否分析舌质舌苔 |
| 辨证逻辑 | 25% | 检查推理过程是否正确 |

### 5.2 实现方案

```typescript
interface AnalysisScorer {
  inputText: string;            // 用户论述文本
  syndromeType: string;         // 正确证型

  // 关键词库
  symptomKeywords: string[];    // 症状关键词
  pulseKeywords: string[];      // 脉象关键词
  tongueKeywords: string[];     // 舌象关键词
  logicKeywords: string[];      // 逻辑关键词

  // 评分方法
  calculateScore(): AnalysisScoreResult;
}

interface AnalysisScoreResult {
  symptomCoverage: number;      // 症状覆盖得分(0-30)
  pulseAnalysis: number;        // 脉象分析得分(0-25)
  tongueAnalysis: number;       // 舌象分析得分(0-20)
  logicScore: number;           // 逻辑得分(0-25)
  totalScore: number;           // 总分(0-100)
  feedback: string;             // 反馈建议
}
```
